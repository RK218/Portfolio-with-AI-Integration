How I Used AI to Design and Ship My Portfolio — Fast

When I decided to apply for this role, I didn't open Figma first. I opened Claude.

My challenge was straightforward but real: I had a working HTML portfolio, but it looked like every other developer's GitHub page — functional, not intentional. I needed to transform it into something that communicated design thinking, not just technical ability.

Here's the workflow I ran:

1. Design Direction with Claude — I fed Claude my existing HTML code and described the role I was targeting. Instead of tweaking colors, we had a conversation about editorial aesthetics, typographic hierarchy, and what visual language would signal a design-aware engineer to a hiring team.

2. Rapid Iteration — What would have taken a full day of trial-and-error in CSS took about 90 minutes. Claude generated the full redesigned portfolio with hover states, sticky navigation, animated project cards, and structured sections.

3. Content & UX Writing — I used Claude to refine the copy across sections, tightening project descriptions and restructuring the experience section so it reads as a growth narrative rather than a bullet dump.

The result: a portfolio that went from a plain HTML file to a structured, typographically intentional, mobile-responsive design — in an afternoon.

What I took away: AI tools don't replace design judgment. They compress the time between idea and validation so you can spend more cycles on decisions that matter — hierarchy, narrative, and communication.

That's exactly the kind of AI-accelerated workflow I'm excited to bring to o9.
